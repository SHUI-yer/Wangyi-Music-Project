# 🎤 实训项目答辩报告 - 网抑云深夜emo (沉浸式音乐播放器)

## 1. 项目简介
本项目基于 Vue 3 + Vite + Pinia 开发，旨在打造一个高标准、沉浸式的本地音乐 Web 播放器。项目不仅还原了经典的网易云微灰度+磨砂质感视觉体验，更集成了沉浸式黑胶播放页、深夜电台(私人FM)、智能元数据解析与跨页面收藏等工业级业务模块。

## 2. 核心技术亮点 (重点展示)

### 🚀 工业级音频播放引擎 (Audio Engine)
- **单例模式**：全局唯一 `Audio` 实例，确保切换页面播放不中断。
- **单向数据流**：严格遵循 `UI -> Store -> Engine` 的调用链，解决了状态不一致与死循环问题。
- **HMR风暴熔断**：针对 Vite 热更新环境，实现了 HMR 安全的 `watchersInitialized` 单例管理，并加入了错误重试次数熔断机制，防止内存爆掉导致闪退。

### 🎨 沉浸式体验与 UI 交互
- **仿黑胶全屏界面**：实现了高级感拉满的旋转唱片、唱针动画以及带透明度的毛玻璃背景。
- **跨页面红心点赞**：全局维护 `favorites` 队列，一键点赞多端同步（播放条、首页、最近播放、搜索页联动）。
- **交互细节**：
  - 进度条拖拽防抖 (Dragging Freeze) 与物理 Audio 时间的强制同步。
  - 图片加载失败的死循环阻断处理 (`errorHandled` 锁)。

### 🧬 智能 Node.js 媒体解析与架构解耦
- **配置驱动架构**：首页分类与歌单详情页的路由映射、标题文案等被彻底抽离至外部 `playlistConfig.js` 配置文件。通过简单的配置修改即可实现全站文案更新与分类扩容，大幅提升了项目的可维护性与扩展性。
- **元数据提取**：引入 `music-metadata` 在后端读取 MP3 文件的 ID3 标签（包括标题、歌手、专辑、内置封面图片）。
- **中文乱码自愈**：利用 `jschardet` 进行编码推断，结合 `iconv-lite` 对老旧的 GBK 编码进行转码修复。

## 3. 业务功能完备性
- **页面数量**：包括本地音乐、下载管理、最近播放、我喜欢的音乐、私人FM等核心页。
- **数据管理**：通过 Pinia 实现播放列表、红心收藏、历史记录的全局共享与持久化。
- **大文件串行上传**：支持大体积（50MB+）音频的无闪退稳定导入，自动落盘至服务器的 `public/media/local` 文件夹。

## 4. 遇到的困难与解决方案
- **Q: 批量上传本地音乐时浏览器为何崩溃？**
  - **A**: 原因是 `Promise.all` 导致的内存瞬间溢出，改用 `for...of` 串行上传后问题解决，并增加了平滑进度条。
- **Q: 播放器无限重复播放暂停导致爆内存闪退怎么办？**
  - **A**: 发现是由于 `useAudio` 在多组件重复引入，导致挂载了多个冲突的 Watcher。一个触发 `play`，另一个捕获到 `AbortError` 后错误暂停并触发切歌，形成指数级死循环。解决方案是引入单例锁，并静默忽略 `AbortError`。
- **Q: 本地歌曲导入全是乱码怎么处理？**
  - **A**: 编写了专门的 Node.js 接口，使用 `jschardet` 判断编码，如果是 `windows-1252` 等会被强制转码为 `GBK` 解码回 `UTF-8`。

## 5. 项目总结
通过本次实训，不仅掌握了 Vue 3 的 Composition API，更深入理解了复杂单页应用 (SPA) 中的状态同步、原生 Audio 调度、Node.js 文件流处理以及内存优化机制，具备了交付准企业级前端项目的能力。
