# 🎤 实训项目答辩报告 - 网易云音乐仿站 (Vue 3)

## 1. 项目简介
本项目基于 Vue 3 + Vite + Pinia 开发，旨在高度还原网易云音乐桌面端的视觉体验与交互逻辑。项目涵盖了发现音乐、歌单详情、播放控制、朋友动态、个人中心等 10+ 个核心业务模块。

## 2. 核心技术亮点 (重点展示)

### 🚀 工业级音频播放引擎 (Audio Engine)
- **单例模式**：全局唯一 `Audio` 实例，确保切换页面播放不中断。
- **单向数据流**：严格遵循 `UI -> Store -> Engine` 的调用链，解决了状态不一致与死循环问题。
- **内存优化**：针对 Vite 热更新环境，实现了 HMR 安全的单例管理，防止内存泄漏。

### 🎨 经典视觉还原与 UI 交互
- **配色规范**：1:1 还原网易红 (#C20C0C) 与侧边栏灰色体系。
- **交互细节**：
  - 进度条拖拽防抖 (Dragging Freeze)。
  - 音量记忆与平滑静音。
  - 播放模式（循环/随机/单曲）深度联动。

## 3. 业务功能完备性
- **页面数量**：12 个功能页。
- **数据管理**：通过 Pinia 实现播放列表、历史记录、用户偏好等状态的全局共享。
- **本地化支持**：支持 20 首本地 MP3 资源的自动映射与连播。

## 4. 遇到的困难与解决方案
- **Q: 如何解决热更新导致的内存占满？**
  - **A**: 发现重复创建 Audio 对象后，将实例挂载到全局 `window` 对象，并在初始化时增加 `isInitialized` 锁。
- **Q: 播放器出现快速连点（播放/停止切换）怎么办？**
  - **A**: 识别出双向绑定导致的反馈环，通过移除 Audio 原生事件对 Store 的反向修改，确立了纯净的“Store 驱动”逻辑。

## 5. 项目总结
通过本次实训，不仅掌握了 Vue 3 的 Composition API 模式，更深入理解了复杂单页应用 (SPA) 中的状态管理与资源调度机制，具备了交付准企业级前端项目的能力。
